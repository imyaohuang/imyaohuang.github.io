# Repository Guidelines

## Purpose

This repository contains Yao Huang's simple academic personal website, published through GitHub Pages at https://yaohuang.org.

## Current Stage

The website is not yet a formal job-market website. It should primarily serve researchers, faculty, students, seminar participants, collaborators, and others who want to learn about Yao Huang's research, teaching, CV, and contact information. It should be easy to convert into a job-market website after August 2026.

## Technical Rules

- Keep the website as a simple static site using plain HTML and CSS.
- Do not add a framework, package manager, custom deployment workflow, analytics, trackers, forms, or unnecessary JavaScript.
- Preserve `CNAME` exactly as it is.
- Preserve existing PDF files and their current public paths unless explicitly told otherwise.
- Do not modify GitHub Pages settings or external DNS.
- Do not publish private, unfinished, or unapproved documents.
- Do not invent academic information, statuses, abstracts, links, or biographical facts.
- Use relative paths for internal pages and files.
- Keep the site responsive, accessible, fast, and easy to maintain.

## Design Rules

- Simple, restrained, academic, and visitor-oriented.
- Use generous whitespace and readable typography.
- No gradients, animations, decorative illustrations, oversized cards, or startup-style visual design.
- Avoid a templated or AI-generated appearance.
- LinkedIn may be included as a secondary professional link.
- Do not include GitHub or Google Scholar links unless explicitly requested later.
- Do not label any paper as a Job Market Paper on the website yet.

## Workflow Rules

- Before a large structural or visual change, propose a plan.
- For routine implementation within an approved plan, proceed without repeatedly asking.
- Run reasonable local checks after changes.
- Do not commit or push unless explicitly instructed.
